# AgentTask Protocol | AI代理任务交付开放协议

## 协议简介
AgentTask Protocol 是一套**去中心化、AI友好、隐私优先**的开放任务交付协议规范，为AI Agent自动接单、结构化交付、自动化验收提供统一的行业标准，填补了国内众包/外包市场无标准化API接口的空白。

本协议**不是平台、不碰资金、不做运营**，只定义任务从发布到交付验收的全流程结构化标准，让任何个人、团队、AI Agent都能基于本协议实现点对点的任务协作，无需依赖中心化平台。

## 核心价值
- 🤖 **AI原生友好**：全结构化JSON格式，可直接被AI Agent读取、解析、自动执行
- 🔒 **隐私优先**：原生支持私密交付规范，交付内容仅任务双方可见，杜绝公开泄露
- 🚀 **去中心化**：无中心平台依赖，可基于飞书、微信群、GitHub、API等任意载体实现
- ⚡ **极简兼容**：轻量无侵入，可兼容现有外包、众包、灵活用工场景，零成本落地

## 适用场景
- AI Agent自动接单/交付/验收全流程自动化
- 个人/小团队去中心化私单协作
- 企业内部零工任务社会性实验
- 垂直领域外包任务标准化流转
- 开发者搭建自动化任务工具

## 快速开始
1. 阅读 [完整协议规范](./SPEC.md) 了解字段定义与流程规范
2. 直接使用 [schema 文件夹](./schema/) 内的JSON标准进行开发
3. 参考 [飞书机器人落地示例](./examples/feishu-bot-demo.md) 零成本搭建自己的自动化任务系统
4. 基于本协议实现的项目，需在显著位置标注「基于 AgentTask Protocol 开发」

## 核心协议结构
任务发布 (Task) → 接单执行 → 私密交付 (Submit) → 自动化验收 (Review) → 任务完结
3个核心模块的完整格式定义，详见 [SPEC.md](./SPEC.md) 与 [schema 规范](./schema/)
