# AgentTask Protocol 完整规范 v1.0.0

## 1. 协议目标
本协议旨在定义一套统一、结构化、AI可识别的任务交付标准，解决当前任务协作场景中：
- 自然语言任务描述无法被AI Agent自动解析
- 交付格式混乱，无法实现自动化验收
- 私密交付无统一规范，内容泄露风险高
- 不同平台/工具之间任务数据无法互通
等核心痛点，为AI时代的去中心化任务协作提供底层标准。

## 2. 术语定义
| 术语 | 定义 |
|------|------|
| 发布者 | 发布任务、支付报酬的一方 |
| 执行者 | 承接任务、完成交付的一方/AI Agent |
| 任务ID | 任务的唯一标识，全协议通用 |
| 私密交付 | 交付内容仅发布者与执行者可见，不对外公开 |
| 自动化验收 | 通过AI/预设规则自动判断交付内容是否符合要求 |

## 3. 设计原则
1. **AI原生优先**：所有字段结构化、标准化，可直接被大语言模型解析执行
2. **去中心化**：不依赖任何中心平台，可在任意载体上实现
3. **隐私优先**：原生支持私密交付，不强制内容公开
4. **极简兼容**：核心字段精简，可扩展，不做强制冗余限制
5. **合规安全**：不涉及支付、清算、金融相关规范，仅定义任务流转标准

## 4. 核心模块规范

---

### 4.1 任务发布规范 (Task)
任务发布的唯一结构化标准，所有字段可直接被AI Agent读取解析

#### 基础结构
| 字段名 | 类型 | 必填 | 说明 | 格式示例 |
|--------|------|------|------|----------|
| protocol | string | 是 | 协议版本号，固定格式 | `AgentTask/v1.0.0` |
| taskId | string | 是 | 任务唯一ID，全局唯一 | `T_20240520_001` |
| title | string | 是 | 任务标题，简洁明了 | `整理3个周口本地免费公园信息` |
| description | string | 否 | 任务详细描述 | `需包含公园地址、开放时间、核心特色` |
| requirements | string | 是 | 交付要求，AI验收核心依据 | `1. 结构化文本输出 2. 信息真实有效 3. 3个公园均为免费开放` |
| reward | number | 是 | 任务报酬金额 | `20` |
| currency | string | 是 | 报酬币种，默认CNY | `CNY` |
| deadline | string | 是 | 任务截止时间，ISO8601格式 | `2024-05-20T18:00:00+08:00` |
| submitType | string | 是 | 交付方式，枚举值 | `private`(私密)/`public`(公开) |
| status | string | 是 | 任务状态，枚举值 | `pending`(待接单)/`assigned`(已接单)/`submitted`(已交付)/`completed`(已完成)/`closed`(已关闭) |
| createdAt | string | 是 | 任务创建时间，ISO8601格式 | `2024-05-20T12:00:00+08:00` |
| updatedAt | string | 否 | 任务更新时间，ISO8601格式 | `2024-05-20T12:30:00+08:00` |
| ext | object | 否 | 扩展字段，可自定义任意内容 | `{"skill": "文案整理", "location": "周口"}` |

#### 任务状态流转规范
1. 任务创建 → 初始状态：`pending`(待接单)
2. 执行者确认接单 → 状态更新为：`assigned`(已接单)
3. 执行者提交交付内容 → 状态更新为：`submitted`(已交付)
4. 验收通过，任务完结 → 状态更新为：`completed`(已完成)
5. 任务取消/过期 → 状态更新为：`closed`(已关闭)

---

### 4.2 任务交付规范 (Submit)
执行者提交交付内容的统一结构化标准，支持私密交付

#### 基础结构
| 字段名 | 类型 | 必填 | 说明 | 格式示例 |
|--------|------|------|------|----------|
| protocol | string | 是 | 协议版本号，与任务对应 | `AgentTask/v1.0.0` |
| submitId | string | 是 | 交付唯一ID | `S_20240520_001` |
| taskId | string | 是 | 对应任务的唯一ID，必须与任务一致 | `T_20240520_001` |
| submitter | string | 是 | 执行者标识，可匿名/哈希值/昵称 | `zhangsan_123` |
| content | string | 是 | 交付核心内容，文本/链接/加密内容 | `1. 周口公园：地址XXX，开放时间XXX...` |
| contentType | string | 是 | 内容类型，枚举值 | `text`/`markdown`/`file`/`link`/`encrypted` |
| timestamp | string | 是 | 交付提交时间，ISO8601格式 | `2024-05-20T17:30:00+08:00` |
| ext | object | 否 | 扩展字段 | `{"fileHash": "xxx", "encryptionType": "AES"}` |

#### 私密交付规范
1. 当任务`submitType`为`private`时，交付内容必须仅对发布者与执行者可见，禁止公开发布
2. 支持加密交付：`content`字段可填写加密后的内容，仅双方持有解密密钥
3. 交付载体推荐：私聊消息、加密文档、私密云存储，禁止在公开群/公开页面提交私密交付内容

---

### 4.3 任务验收规范 (Review)
验收结果的统一结构化标准，支持AI自动化验收

#### 基础结构
| 字段名 | 类型 | 必填 | 说明 | 格式示例 |
|--------|------|------|------|----------|
| protocol | string | 是 | 协议版本号，与任务对应 | `AgentTask/v1.0.0` |
| reviewId | string | 是 | 验收唯一ID | `R_20240520_001` |
| taskId | string | 是 | 对应任务的唯一ID | `T_20240520_001` |
| submitId | string | 是 | 对应交付的唯一ID | `S_20240520_001` |
| result | string | 是 | 验收结果，枚举值 | `pass`(通过)/`reject`(驳回)/`revise`(需修改) |
| comment | string | 是 | 验收评语，驳回/需修改时必须填写原因 | `内容完整，符合要求，验收通过` |
| automated | boolean | 是 | 是否为AI自动化验收 | `true`/`false` |
| reviewer | string | 是 | 验收人标识 | `task_publisher`/`ai_agent` |
| timestamp | string | 是 | 验收时间，ISO8601格式 | `2024-05-20T17:40:00+08:00` |
| ext | object | 否 | 扩展字段 | `{"aiModel": "doubao-4", "confidence": 0.98}` |

---

## 5. 兼容性说明
- 本协议向下兼容，所有`v1.x.x`版本的核心字段完全兼容
- 扩展字段`ext`可根据不同场景自定义，不影响核心流程
- 可基于本协议适配飞书、企业微信、钉钉、API接口、网页等任意载体

## 6. 合规要求
1. 所有基于本协议的任务内容，必须遵守中华人民共和国法律法规
2. 禁止发布违法违规、侵犯他人权益、危害国家安全的任务
3. 禁止利用本协议从事洗钱、非法集资、虚假交易等违法金融活动
4. 所有使用者需自行承担任务相关的法律责任与风险
